# stack-spec.yaml Reference

> **Version:** 1.0  
> **Format:** YAML (standardized)  
> **Purpose:** Standardized input for StackKits (output from KombiStack Unifier or manual creation)

## Overview

`stack-spec.yaml` is the **standardized spec format** that StackKits consume. It contains fully resolved configuration ready for infrastructure provisioning.

**Two Ways to Create:**

1. **KombiStack Users:** Auto-generated by Unifier from `kombination.yaml` (user intent)
2. **CLI-Only Users:** Manually create or use `default-spec.yaml` templates

## Format Structure

### Complete Schema

```yaml
# stack-spec.yaml - Standardized StackKit Input
version: "1.0"  # Spec version

stack:
  name: string           # Stack identifier (DNS-compatible)
  kit: string            # StackKit name (e.g., "base-homelab")
  variant: string        # Variant path (e.g., "os/ubuntu-24")
  addons: [string]       # Add-on list (e.g., ["monitoring"])

nodes:
  - name: string         # Node identifier
    type: string         # "main" | "worker"
    provider: string     # "local" | "hetzner" | "docker"
    ip: string           # Node IP address
    ssh:
      host: string       # SSH hostname/IP
      port: int          # SSH port (default: 22)
      user: string       # SSH username
      key_path: string   # Path to SSH private key
    os:
      type: string       # "ubuntu" | "debian" | "fedora"
      version: string    # OS version (e.g., "24.04")
      arch: string       # "amd64" | "arm64"
    compute:
      tier: string       # "low" | "standard" | "high"
      min_ram_gb: int    # Minimum RAM (GB)
      min_cpu_cores: int # Minimum CPU cores
    docker:              # For Docker-based StackKits
      version: string    # Docker version
      installed: bool    # Pre-installed?
    k3s:                 # For Kubernetes-based StackKits
      role: string       # "server" | "agent"
      version: string    # K3s version
      options: [string]  # CLI flags

services:
  - name: string         # Service identifier
    type: string         # Service type (reverse-proxy, database, etc.)
    node: string         # Target node name
    image: string        # Container image (fully qualified)
    ports: [int]         # Exposed ports
    volumes: [string]    # Volume mounts (host:container:mode)
    env:                 # Environment variables
      KEY: value
    labels:              # Container/Pod labels
      key: value
    needs: [string]      # Dependencies (service names)
    deploy:              # Deployment config (Docker Swarm/K8s)
      mode: string       # "replicated" | "global"
      replicas: int
      placement:
        constraints: [string]
    k8s:                 # Kubernetes-specific
      namespace: string
      replicas: int
      ingress:
        enabled: bool
        host: string
        tls:
          enabled: bool

network:
  mode: string           # "local" | "public" | "hybrid"
  docker_network: string # Docker network name
  subnet: string         # Network subnet (CIDR)
  gateway: string        # Gateway IP
  driver: string         # "bridge" | "overlay"
  k8s_service_cidr: string  # K8s service CIDR
  k8s_pod_cidr: string      # K8s pod CIDR
  ingress_ip: string        # External ingress IP
  dns:
    provider: string     # DNS provider (cloudflare, route53)
    email: string        # ACME email

tls:                     # TLS configuration (public mode)
  enabled: bool
  provider: string       # "letsencrypt" | "self-signed"
  email: string          # ACME email
  staging: bool          # Use staging CA?

metadata:
  created_at: string     # ISO 8601 timestamp
  description: string    # Human-readable description
  unifier_version: string  # Unifier version (KombiStack) or "n/a"
  stackkit_version: string # StackKit version
  tags: [string]         # Arbitrary tags
```

## Examples

### Example 1: base-homelab (Single Server)

```yaml
version: "1.0"

stack:
  name: my-homelab
  kit: base-homelab
  variant: os/ubuntu-24
  addons: []

nodes:
  - name: homelab-server
    type: main
    provider: local
    ip: 192.168.1.100
    ssh:
      host: 192.168.1.100
      port: 22
      user: admin
      key_path: ~/.ssh/id_ed25519
    os:
      type: ubuntu
      version: "24.04"
      arch: amd64
    compute:
      tier: standard
      min_ram_gb: 4
      min_cpu_cores: 2
    docker:
      version: "27.5.1"
      installed: false

services:
  - name: traefik
    type: reverse-proxy
    node: homelab-server
    image: traefik:v3.2
    ports: [80, 443, 8080]
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
    env:
      TRAEFIK_API_DASHBOARD: "true"
    labels:
      kombistack.service: traefik

  - name: dokploy
    type: deployment-platform
    node: homelab-server
    image: dokploy/dokploy:latest
    ports: [3000]
    needs: [postgres]

network:
  mode: local
  docker_network: kombi_net
  subnet: 172.20.0.0/16

metadata:
  created_at: "2026-01-10T12:00:00Z"
  stackkit_version: "1.0.0"
```

### Example 2: ha-homelab (3-Node Docker Swarm)

```yaml
version: "1.0"

stack:
  name: ha-homelab
  kit: ha-homelab
  variant: os/ubuntu-24
  addons: [monitoring]

nodes:
  - name: homelab-main
    type: main
    provider: local
    ip: 192.168.1.100
    docker:
      swarm:
        role: manager
        advertise_addr: 192.168.1.100
  
  - name: homelab-worker-1
    type: worker
    ip: 192.168.1.101
    docker:
      swarm:
        role: worker
  
  - name: homelab-worker-2
    type: worker
    ip: 192.168.1.102
    docker:
      swarm:
        role: worker

services:
  - name: traefik
    type: reverse-proxy
    node: homelab-main
    deploy:
      mode: replicated
      replicas: 1
      placement:
        constraints:
          - node.role == manager

  - name: dokploy
    type: deployment-platform
    node: homelab-worker-1
    deploy:
      mode: replicated
      replicas: 2
      placement:
        constraints:
          - node.role == worker

network:
  mode: local
  docker_network: kombi_overlay
  subnet: 10.0.1.0/24
  driver: overlay
```

### Example 3: modern-homelab (Kubernetes)

```yaml
version: "1.0"

stack:
  name: modern-homelab
  kit: modern-homelab
  variant: os/ubuntu-24
  addons: [monitoring, observability]

nodes:
  - name: k3s-control
    type: main

    k3s:
      role: server
      version: "v1.31.4+k3s1"
      options:
        - "--disable=traefik"
        - "--tls-san=192.168.1.100"
  
  - name: k3s-worker-1
    type: worker
    ip: 192.168.1.101
    k3s:
      role: agent
  
  - name: k3s-worker-2
    type: worker
    ip: 192.168.1.102
    k3s:
      role: agent

services:
  - name: ingress-nginx
    type: ingress
    k8s:
      namespace: ingress-nginx
      install_method: helm
      chart: ingress-nginx/ingress-nginx

  - name: dokploy
    type: deployment-platform
    k8s:
      namespace: default
      replicas: 2
      ingress:
        enabled: true
        host: dokploy.yourdomain.com
        tls:
          enabled: true
          issuer: letsencrypt-prod

network:
  mode: public
  k8s_service_cidr: 10.43.0.0/16
  k8s_pod_cidr: 10.42.0.0/16
  ingress_ip: 192.168.1.100

tls:
  enabled: true
  provider: letsencrypt
  email: admin@yourdomain.com
```

## Field Descriptions

### Stack Fields

| Field | Required | Description |
|-------|----------|-------------|
| `version` | ✅ | Spec format version (currently "1.0") |
| `stack.name` | ✅ | Stack identifier (DNS-compatible, lowercase) |
| `stack.kit` | ✅ | StackKit name (must exist in repo) |
| `stack.variant` | ✅ | Variant path (e.g., "os/ubuntu-24") |
| `stack.addons` | ❌ | Add-ons to enable (e.g., ["monitoring"]) |

### Node Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | ✅ | Node identifier (unique within stack) |
| `type` | ✅ | Node role: "main" (control plane) or "worker" |
| `provider` | ✅ | Infrastructure provider (local/hetzner/docker) |
| `ip` | ✅ | Node IP address (reachable from provisioning host) |
| `ssh.host` | ✅ | SSH hostname (usually same as `ip`) |
| `ssh.user` | ✅ | SSH username |
| `ssh.key_path` | ✅ | Path to SSH private key |
| `os.type` | ✅ | OS family (ubuntu/debian/fedora) |
| `os.version` | ✅ | OS version (e.g., "24.04") |
| `compute.tier` | ✅ | Resource tier (low/standard/high) |
| `docker.version` | Docker only | Docker version to install |
| `k3s.role` | K8s only | K3s role (server/agent) |

### Service Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | ✅ | Service identifier (DNS-compatible) |
| `type` | ✅ | Service category (reverse-proxy/database/etc.) |
| `node` | ✅ | Target node name (or "any" for K8s) |
| `image` | ✅ | Container image (fully qualified) |
| `ports` | ❌ | Exposed ports (array of integers) |
| `volumes` | ❌ | Volume mounts (host:container:mode) |
| `env` | ❌ | Environment variables (key-value map) |
| `labels` | ❌ | Container/Pod labels (key-value map) |
| `needs` | ❌ | Service dependencies (array of service names) |
| `deploy` | HA only | Swarm deployment config |
| `k8s` | K8s only | Kubernetes-specific config |

### Network Fields

| Field | Required | Description |
|-------|----------|-------------|
| `mode` | ✅ | Network mode (local/public/hybrid) |
| `docker_network` | Docker | Docker network name |
| `subnet` | ❌ | Network subnet (CIDR notation) |
| `driver` | HA | Network driver (bridge/overlay) |
| `k8s_service_cidr` | K8s | K8s service CIDR |
| `ingress_ip` | Public | External ingress IP |

## Validation Rules

### Stack Validation

- `stack.name` must match `^[a-z][a-z0-9-]+$` (DNS-compatible)
- `stack.kit` must reference an existing StackKit
- `stack.variant` must exist in the StackKit's variants/

### Node Validation

- At least one node with `type: main` required
- Node names must be unique within stack
- IPs must be valid IPv4 addresses
- SSH configuration must be complete (host, user, key_path)

### Service Validation

- Service names must be unique within stack
- `service.node` must reference an existing node (or "any" for K8s)
- `service.needs` must reference existing services (no circular deps)
- Ports must be in range 1-65535

### Network Validation

- `mode: public` requires `tls.enabled: true`
- `driver: overlay` requires Docker Swarm (multiple nodes)
- K8s CIDRs must not overlap with node IPs

## Differences from kombination.yaml

| Aspect | kombination.yaml | stack-spec.yaml |
|--------|------------------|-----------------|
| **Purpose** | User intent (what) | Execution config (how) |
| **Created by** | UI Wizard | Unifier or manual |
| **Resolved** | ❌ Abstract | ✅ Fully resolved |
| **IPs/Ports** | ❌ Not specified | ✅ All specified |
| **Defaults** | ❌ Not applied | ✅ Applied |
| **Validation** | ✅ Pre-validation | ✅ Final validation |
| **StackKit Input** | ❌ No | ✅ Yes |
| **Modification** | ✅ User edits | ❌ Immutable (regenerate) |

## Usage

### With KombiStack

```bash
# User creates kombination.yaml via UI
# Unifier generates stack-spec.yaml automatically
kombistack apply my-stack
```

### CLI-Only (Standalone)

```bash
# Start from template
cp base-homelab/default-spec.yaml my-stack-spec.yaml

# Edit configuration
nano my-stack-spec.yaml

# Validate
stackkit validate my-stack-spec.yaml

# Deploy
stackkit apply my-stack-spec.yaml
```
