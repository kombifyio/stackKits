# stack-spec.yaml Reference

> **Version:** 2.0 (V5 Architecture)
> **Format:** YAML
> **Purpose:** Standardized deployment input for StackKits

## Overview

`stack-spec.yaml` is the deployment specification that StackKits consume. It describes WHAT to deploy, WHERE, and HOW — the CLI then generates all infrastructure artifacts from it.

**Two Ways to Create:**

1. **kombify Stack Users:** Auto-generated by Unifier from `kombination.yaml`
2. **CLI-Only Users:** Create manually or use `stackkit init`

## Schema

```yaml
# stack-spec.yaml — StackKit Deployment Specification
name: string              # Stack identifier (DNS-compatible, e.g. "mylab")
stackkit: string          # StackKit name: "base-kit", "modern-homelab", "ha-kit"
mode: string              # Deployment engine: "simple" (OpenTofu) or "advanced" (Terramate)
runtime: string           # "docker" (default) or "native"
context: string           # Auto-detected: "local", "cloud", "pi" (rarely set manually)

# Domain & Access
domain: string            # Domain for service URLs (see Domain Modes below)
subdomainPrefix: string   # For kombify.me: prefix for subdomain (e.g. "mylab")
email: string             # Admin email (shown in UI, used for ACME)
adminEmail: string        # ACME registration email (falls back to email)

# TLS Certificate Configuration
tls:
  provider: string        # DNS provider for DNS-01 challenge: "cloudflare", "hetzner", "digitalocean", "duckdns", "namecheap"
  challenge: string       # "tls" (TLS-ALPN-01, default) or "dns" (DNS-01, NAT-friendly)

# Network
network:
  mode: string            # "bridge" (default) or "host"
  subnet: string          # Docker network CIDR (default: "172.20.0.0/16")

# Compute
compute:
  tier: string            # "low", "standard" (default), "high"

# PAAS Platform (optional — auto-detected from compute tier if omitted)
paas: string              # "dokploy" (default for standard/high), "coolify", "dockge" (default for low), "none"

# Storage (optional — for devices with limited disk)
storage:
  externalDevice: string  # Block device path (e.g. "/dev/sdb")
  mountPoint: string      # Mount point (e.g. "/mnt/data")

# SSH
ssh:
  user: string            # SSH username
  port: int               # SSH port (default: 22)

# Nodes
nodes:
  - name: string          # Node identifier
    role: string          # "standalone", "main", "worker"
    ip: string            # Node IP address

# Add-ons
addons: [string]          # Add-on names (e.g. ["monitoring", "vpn-overlay"])

# Service overrides (advanced)
services:
  <name>: any             # Per-service config overrides

# Environment variables passed to deployment
environment:
  KEY: value
```

## Domain Modes

StackKits support three domain access modes. Each determines how service URLs are generated, how TLS certificates are obtained, and how DNS resolution works.

### Mode 1: Custom Domain

```yaml
domain: kmbchr.de
email: admin@kmbchr.de
tls:
  provider: cloudflare    # Required if device is behind NAT
```

- **Service URLs:** `service.kmbchr.de` (e.g. `base.kmbchr.de`, `kuma.kmbchr.de`)
- **TLS:** Let's Encrypt via ACME. TLS-ALPN-01 (default, needs public port 443) or DNS-01 (NAT-friendly, needs DNS provider API)
- **DNS:** User manages DNS records (wildcard `*.kmbchr.de` → server IP)
- **Environment variables for DNS-01:**
  - `STACKKIT_DNS_TOKEN` — API key/token for the DNS provider
  - `STACKKIT_DNS_EMAIL` — Email for Cloudflare Global API Key auth (optional for scoped tokens)

### Mode 2: kombify.me

```yaml
domain: kombify.me
subdomainPrefix: mylab
```

- **Service URLs:** `mylab-service.kombify.me` (flat naming, single DNS level)
- **TLS:** Managed by kombify (Cloudflare wildcard)
- **DNS:** Automatic via kombify.me subdomain registry
- **Requires:** `KOMBIFY_API_KEY` environment variable
- **Direct Connect:** After deploy, `stackkit apply` registers the instance with kombify. `stackkit-server` sends heartbeats so Kong can proxy traffic.

### Mode 3: Local (home.lab)

```yaml
domain: home.lab
# or omit domain entirely (defaults to home.lab)
```

- **Service URLs:** `service.home.lab` (e.g. `auth.home.lab`, `kuma.home.lab`)
- **TLS:** Self-signed (no ACME — local network only)
- **DNS:** dnsmasq container resolves `*.home.lab` → server LAN IP
- **Note:** Domain must be `home.lab` (not `homelab`) — TinyAuth requires second-level domain

## TLS Challenge Types

| Challenge | Flag | When to Use | Requirements |
|-----------|------|-------------|--------------|
| TLS-ALPN-01 | `tls` (default) | Server has public IP, port 443 reachable | Public port 443 |
| DNS-01 | `dns` | Behind NAT, local network, wildcard certs | DNS provider API credentials |

DNS-01 is auto-selected when `tls.provider` is set. Supported providers:

| Provider | `tls.provider` value | Environment Variables |
|----------|---------------------|----------------------|
| Cloudflare | `cloudflare` | `STACKKIT_DNS_TOKEN` + `STACKKIT_DNS_EMAIL` (Global API Key) or `STACKKIT_DNS_TOKEN` only (scoped API Token) |
| Hetzner | `hetzner` | `STACKKIT_DNS_TOKEN` |
| DigitalOcean | `digitalocean` | `STACKKIT_DNS_TOKEN` |
| DuckDNS | `duckdns` | `STACKKIT_DNS_TOKEN` |
| Namecheap | `namecheap` | `STACKKIT_DNS_TOKEN` + `STACKKIT_DNS_EMAIL` |

## Compute Tiers

| Tier | Criteria | Effect |
|------|----------|--------|
| `high` | 8+ CPU, 16+ GB RAM | Full stack, all use cases |
| `standard` | 4+ CPU, 4+ GB RAM | Default. Dokploy + PostgreSQL + Redis |
| `low` | <4 CPU or <4 GB RAM | Dockge replaces Dokploy. No heavy services |

## PAAS Platforms

| Platform | When Used | Reverse Proxy |
|----------|-----------|---------------|
| `dokploy` | Default for standard/high tier | Dokploy's Traefik (platform services attach to Dokploy's network) |
| `coolify` | Explicit `paas: coolify` | Coolify's Traefik (platform services attach to Coolify's network) |
| `dockge` | Default for low tier | Standalone Traefik (StackKit deploys its own) |
| `none` | No PAAS | Standalone Traefik |

When `paas` is omitted, it is auto-detected from the compute tier:
- `standard` / `high` → `dokploy`
- `low` → `dockge`

**Reverse proxy backend:** When Dokploy or Coolify is the PAAS, they manage their own Traefik instance. Platform services (TinyAuth, PocketID, Dashboard, Kuma, Whoami) attach to the PAAS Traefik network for routing. StackKit does NOT deploy a separate Traefik in this case.

When Dockge is the PAAS (or no PAAS), StackKit deploys its own standalone Traefik container.

See [ADR-0006](ADR/ADR-0006-service-url-matrix.md) for the full 9-scenario Service URL Matrix.

## Examples

### Base Kit — Custom Domain (behind NAT)

```yaml
name: mylab
stackkit: base-kit
mode: simple
domain: example.com
email: admin@example.com
tls:
  provider: cloudflare
network:
  mode: bridge
  subnet: 172.20.0.0/16
compute:
  tier: standard
ssh:
  user: admin
  port: 22
nodes:
  - name: homeserver
    role: standalone
    ip: 192.168.1.100
```

```bash
STACKKIT_DNS_TOKEN=cf_api_key_here \
STACKKIT_DNS_EMAIL=admin@example.com \
stackkit generate && stackkit apply --auto-approve
```

### Base Kit — Local Network

```yaml
name: homelab
stackkit: base-kit
mode: simple
domain: home.lab
compute:
  tier: low
ssh:
  user: pi
  port: 22
nodes:
  - name: rpi
    role: standalone
    ip: 192.168.1.50
```

### Base Kit — kombify.me

```yaml
name: mylab
stackkit: base-kit
mode: simple
domain: kombify.me
subdomainPrefix: mylab
email: user@example.com
compute:
  tier: standard
ssh:
  user: admin
  port: 22
nodes:
  - name: server
    role: standalone
    ip: 10.0.0.5
```

### Base Kit — Coolify PAAS

```yaml
name: mylab
stackkit: base-kit
mode: simple
domain: example.com
email: admin@example.com
paas: coolify
tls:
  provider: cloudflare
compute:
  tier: standard
ssh:
  user: admin
  port: 22
nodes:
  - name: server
    role: standalone
    ip: 10.0.0.5
```

Platform services (TinyAuth, PocketID, Dashboard, Kuma, Whoami) route through Coolify's Traefik — no separate Traefik container is deployed.

## Validation Rules

- `name` must be DNS-compatible: `^[a-z][a-z0-9-]+$`
- `stackkit` must reference an existing StackKit (`base-kit`, `modern-homelab`, `ha-kit`)
- `mode` must be `simple` or `advanced`
- `compute.tier` must be `low`, `standard`, or `high`
- `paas` must be `dokploy`, `coolify`, `dockge`, or `none` (auto-detected if omitted)
- `tls.provider` auto-selects `challenge: dns` if set
- `domain: home.lab` disables HTTPS (self-signed only)
- `domain: kombify.me` requires `subdomainPrefix` and `KOMBIFY_API_KEY`
- At least one node required
- Node IPs must be valid IPv4
